# Retrospective Template

Use this structure for retrospective outputs.

## Scope Scanned

- current repo
- sibling repos searched
- home directories searched
- notable exclusions

## High-Signal Patterns

List the repeated successful moves.

For each:

- pattern
- evidence
- why it matters
- whether it is public, private, or local

## Mistakes And Friction

List repeated failures, confusion, or unnecessary work.

For each:

- issue
- evidence
- likely root cause
- whether a skill, tool, or local process change would help

## Candidate Skills

Use a table:

| Skill | Type | Why now | Public/private/local | Next step |
| --- | --- | --- | --- | --- |

## Keep-Local Items

List findings that belong in the current repo rather than a shared skill.

## Recommended Next Actions

Rank the next 1-3 moves.

## Starter Stubs

Only include this section in `scaffold` mode.
